- [ ] T001 [US4] Add the `blob.Store` interface and `Ref` type at `go/orchestrator/internal/blob/store.go` per FR-001. `Ref` wraps a `blob://sha256/<hex>` URI; its zero value is the empty reference. Methods: `Put(ctx, body) (Ref, error)` and `Get(ctx, ref) ([]byte, error)`. The package doc comment states the inline-vs-externalized contract so a future reader doesn't need the spec to understand the shape
- [ ] T002 [US4] Implement `FSStore` at `go/orchestrator/internal/blob/fsstore.go` per FR-002. Sharded layout `<dir>/<first2hex>/<rest>.blob`; atomic write via tmp-file + fsync + rename; honors `$CHITIN_BLOB_DIR` with default `~/.chitin/blobs/`. Idempotent on identical-content writes (rename-over-existing is a no-op)
- [ ] T003 [US1] Add the `Externalize(ctx, store, body) (string, error)` helper at `go/orchestrator/internal/blob/externalize.go` per FR-004. Bodies Ōēż `InlineThreshold` (1,048,576 bytes) returned verbatim as their string form; larger bodies written via `store.Put`, return value is the `blob://sha256/<hash>` URI string. The threshold is an exported const so tests can override
- [ ] T004 [US3] Add the `Resolve(ctx, store, s) ([]byte, error)` helper at `go/orchestrator/internal/blob/resolve.go` per FR-007. Inputs without the `blob://` prefix returned as `[]byte(s)` (no-op); inputs with the prefix parsed, hash extracted, fetched via `store.Get`. Callers can `Resolve` unconditionally
- [ ] T005 [US4] Add the `S3Store` skeleton at `go/orchestrator/internal/blob/s3store.go` per FR-012. Satisfies the `Store` interface; `Put` / `Get` bodies return `errors.New("s3 blob store not implemented")`. Goal: lock the interface shape for a future spec, NOT ship a working S3 impl. A build-only test asserts the type satisfies the interface
