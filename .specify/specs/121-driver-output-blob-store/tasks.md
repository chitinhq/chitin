- [x] T001 [US4] Add the `blob.Store` interface and `Ref` type at `go/orchestrator/internal/blob/store.go` per FR-001. `Ref` wraps a `blob://sha256/<hex>` URI; its zero value is the empty reference. Methods: `Put(ctx, body) (Ref, error)` and `Get(ctx, ref) ([]byte, error)`. The package doc comment states the inline-vs-externalized contract so a future reader doesn't need the spec to understand the shape
- [x] T002 [US4] Implement `FSStore` at `go/orchestrator/internal/blob/fsstore.go` per FR-002 + FR-003. Content-addressed naming: blob filename is the SHA-256 hex of the body bytes per FR-003 (dedup is free; identical-content Puts produce identical refs). Sharded layout `<dir>/<first2hex>/<rest>.blob`; atomic write via tmp-file + fsync + rename; honors `$CHITIN_BLOB_DIR` with default `~/.chitin/blobs/`. Idempotent on identical-content writes (rename-over-existing is a no-op)
- [x] T003 [US1] Add the `Externalize(ctx, store, body) (string, error)` helper at `go/orchestrator/internal/blob/externalize.go` per FR-004. Bodies ≤ `InlineThreshold` (1,048,576 bytes) returned verbatim as their string form; larger bodies written via `store.Put`, return value is the `blob://sha256/<hash>` URI string. The threshold is an exported const so tests can override
- [x] T004 [US3] Add the `Resolve(ctx, store, s) ([]byte, error)` helper at `go/orchestrator/internal/blob/resolve.go` per FR-007. Inputs without the `blob://` prefix returned as `[]byte(s)` (no-op); inputs with the prefix parsed, hash extracted, fetched via `store.Get`. Callers can `Resolve` unconditionally
- [x] T005 [US4] Add the `S3Store` skeleton at `go/orchestrator/internal/blob/s3store.go` per FR-012. Satisfies the `Store` interface; `Put` / `Get` bodies return `errors.New("s3 blob store not implemented")`. Goal: lock the interface shape for a future spec, NOT ship a working S3 impl. A build-only test asserts the type satisfies the interface
- [x] T006 [US1] Update `driver/claudecode/driver.go:154` (`resultFromCommand`) per FR-005 + FR-008: pass `stdout` through `blob.Externalize` before assigning to `Result.OutputRef`. The `WorkUnitResult` struct shape MUST stay unchanged per FR-008 — the `OutputRef` field continues to carry a string; nothing else moves. Inject a `blob.Store` via the existing driver-construction path (`NewDriver` or the registry) — do NOT introduce a global. A `nil` store falls back to literal-string pass-through so existing tests don't need rewriting
- [x] T007 [US1] Update `driver/codex/driver.go:170` (the build-result analog of claudecode's `resultFromCommand`) per FR-005. Same change: `Externalize` on stdout. Do NOT touch the review-mode `extractStructuredVerdict` path — that output is bounded by the verdict schema and would never need externalization
- [x] T008 [US1] Apply `Externalize` to `Result.Explanation` at both driver sites per FR-006 — covers the case where a crashed driver dumps oversized stderr into the failure-path Explanation. The success-path Explanation is small (~80 bytes) and inline; this is defensive for the failure-path
- [x] T009 [US5] Emit the `blob_written` chain event per FR-010 from `FSStore.Put` (and `S3Store.Put` when impl lands). Payload: `{ref, size_bytes, sha256}`. Emit lives next to the rename — only fires on a successful write, never on a no-op idempotent re-Put (otherwise the chain fills with duplicates for the same content)
