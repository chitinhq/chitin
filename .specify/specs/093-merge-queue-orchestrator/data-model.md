# Phase 1 Data Model: Merge Queue Orchestrator

**Spec**: [spec.md](./spec.md) | **Plan**: [plan.md](./plan.md) | **Research**: [research.md](./research.md)

This document defines every entity the spec names plus the workflow input/output and signal types the implementation needs. Field names use Go conventions (PascalCase) since the implementation is Go-only.

---

## Domain entities (from spec)

### MergeQueueSubmission

A single operator-issued request to merge an ordered list of PRs. One submission produces one `MergeQueueWorkflow` execution.

| Field | Type | Validation | Notes |
|-------|------|-----------|-------|
| `SubmissionID` | `string` (ULID) | required; ULID format | Generated by the CLI at submission time; becomes the parent workflow ID suffix (R-ID). |
| `Label` | `string` | optional; ≤ 200 chars | Human-readable description for operator inspection. |
| `Entries` | `[]QueueEntry` | required; non-empty; ≤ 100 entries | Order is preserved; index in slice = queue position. |
| `SubmittedAt` | `time.Time` | required; set by CLI | UTC. |
| `SubmittedBy` | `string` | required; set by CLI from local user | Audit only; does not affect policy gates (FR-019 no-bypass). |
| `PolicyTableVersion` | `string` | required; semver | The policy table version snapshot the submission is bound to. Set by CLI at submission time. |

**Invariants**:
- All `Entries[i].DependsOn` indices reference earlier positions (`< i`); no cycles (FR-002).
- No duplicate `(Repo, PRNumber)` pairs within Entries (FR-002).
- All `Entries[i].Repo` values must have credentials on the worker (FR-002).

### MergeabilityState (enum)

Mirrors GitHub's `mergeStateStatus` field with normalized names. Workflow routes on this enum, not the raw GitHub string.

| Value | Meaning | Workflow handler |
|-------|---------|------------------|
| `Clean` | Mergeable, all checks pass | Proceed to merge |
| `Unstable` | Mergeable, checks pending or non-required failing | Proceed to check-wait |
| `Behind` | Base moved; rebase needed | Proceed to rebase-with-policy |
| `Dirty` | Conflicts present | Proceed to rebase-with-policy (which decides auto-resolve vs halt) |
| `Blocked` | Approvals/required checks missing | Halt with operator gate |
| `Unknown` | GitHub hasn't computed yet | Backoff + retry within activity |
| `Draft` | PR is a draft | Halt with operator gate |
| `Closed` | PR is closed or merged elsewhere | Skip (FR-014) |

## Workflow inputs and outputs

### `MergeQueueWorkflow` (parent)

- **Input**: `MergeQueueSubmission`
- **Output**: `QueueResult`
- **Workflow ID**: `merge-queue-${SubmissionID}` (R-ID)
- **Task queue**: `"chitin"`

**Behavior** (deterministic — all I/O via activities):

1. For each `Entries[i]` in order:
   1. Check `Entries[i].DependsOn`: every referenced earlier index must have `EntryResult.Status == merged`. If any is `skipped`/`halted`/`aborted`, skip this entry (status = `not-attempted`, reason = "dependency unmet").
   2. Start a child workflow `PRMergeWorkflow` with ID `merge-pr-${Repo}-${PRNumber}` (R-ID). Use `WorkflowIDReusePolicy: REJECT_DUPLICATE`.
   3. Await the child:
      - If the child returns `EntryResult.Status == merged`: record and continue.
      - If the child returns `skipped` (already-merged detection, FR-014): record and continue.
      - If the child returns `halted` or `aborted`: record, set `HaltedAtIndex = i`, set `HaltReason`, mark all remaining entries as `not-attempted`, exit loop.
      - If the child returns "already exists" (because someone else's parent already started a merge for this PR): record `skipped`, reason `"already-running"`, continue.
2. Emit final `QueueResult` to OTLP via `emit_queue_telemetry` activity.
3. Emit Discord notification with summary via `notify_operator` activity.
4. Return `QueueResult`.

---

### `PRMergeWorkflow` (child)

- **Input**: `PRMergeInput`
- **Output**: `EntryResult`
- **Workflow ID**: `merge-pr-${Repo}-${PRNumber}` (R-ID)
- **Task queue**: `"chitin"`
- **Signals accepted**: `resume`, `abort`, `approve` (R-SIG)
- **Queries accepted**: `state` returns `PRMergeState` (current step + last reason)

### PRMergeInput

| Field | Type | Notes |
|-------|------|-------|
| `Repo` | `string` | from QueueEntry |
| `PRNumber` | `int` | from QueueEntry |
| `ExpectedClass` | `*PolicyClass` | from QueueEntry; nil if no override |
| `PolicyTableVersion` | `string` | from MergeQueueSubmission |
| `SubmissionID` | `string` | from MergeQueueSubmission |
| `EntryIndex` | `int` | for telemetry attribution |

### PRMergeState

| Field | Type | Notes |
|-------|------|-------|
| `Step` | `string` | one of: `fetching`, `classifying`, `waiting-approval`, `rebasing`, `pushing`, `waiting-checks`, `merging`, `cleanup`, `done`, `paused`, `failed` |
| `LastReason` | `string` | human-readable last decision |
| `AssignedClass` | `*PolicyClass` | set after classification |
| `LastSnapshot` | `*PRSnapshot` | most recent snapshot |
| `AttemptCount` | `int` | how many times the rebase/check loop has iterated |

**Behavior**:

1. `fetch_pr_metadata` → produces `PRSnapshot`. If PR is closed or merged (FR-014): return `EntryResult{Status: skipped, Reason: "already-merged-or-closed"}`.
2. `classify_pr(snapshot, policyTable)` → produces `PolicyClass`. Compare with `ExpectedClass`; if mismatch and `ExpectedClass` relaxes governance: halt.
3. If `ClassPolicy.RequiresApproval` is true (governance): block on `approve` signal indefinitely. On receipt, continue.
4. **Loop** (max 10 iterations to bound):
   1. `check_mergeability(repo, pr)` → produces fresh `MergeabilityState`.
   2. Branch on state:
      - `Clean`: break loop and proceed to merge.
      - `Unstable`: proceed to `wait_for_checks`. On success, loop again.
      - `Behind` or `Dirty`: proceed to `rebase_with_policy`. On `auto-resolved`, `force_push_with_lease` then loop. On `halt-non-pointer-conflict`, signal-block on `resume` or `abort`. On `resume`, loop. On `abort`, return `EntryResult{Status: aborted}`.
      - `Blocked`, `Draft`: signal-block on `resume` or `abort`.
      - `Closed`: return `EntryResult{Status: skipped, Reason: "closed-during-queue"}`.
      - `Unknown`: workflow-side sleep 5s, loop.
5. After loop break: `merge_pr` (squash + delete-branch). On success: return `EntryResult{Status: merged, MergeSHA, MergedAt, BranchDeleted}`.
6. If `merge_pr` reports branch-not-deleted (rare GitHub timing issue), fire `delete_branch` as cleanup; set `BranchDeleted` from cleanup result.
7. Emit `emit_queue_telemetry` activity on every step transition (FR-024).

## Signal types

### `resume` signal

- **Name**: `resume`
- **Payload**: empty struct `{}`
- **Effect**: if in `paused`/`waiting-approval`/halted-rebase state, transition to re-check mergeability; otherwise no-op (Temporal records signal in history but workflow ignores)

### `abort` signal

- **Name**: `abort`
- **Payload**: `{Reason: string}`
- **Effect**: terminate the workflow with `EntryResult{Status: aborted, Reason: payload.Reason}`; parent halts the queue.

### `approve` signal

- **Name**: `approve`
- **Payload**: `{Approver: string, Note: string}`
- **Effect**: only meaningful in `waiting-approval` step; releases the gate and continues to mergeability check. Recorded in OTLP telemetry as approval event.

## Activity I/O summary

All activities follow the convention `Execute(ctx, TypedInput) (TypedResult, error)` and register with name `merge.<ActivityName>`.

| Activity | Input | Output | Timeout |
|----------|-------|--------|---------|
| `merge.FetchPRMetadata` | `{Repo, PRNumber}` | `PRSnapshot` | 30s |
| `merge.ClassifyPR` | `{Snapshot, TableVersion}` | `{Class, Reason}` | 5s (pure CPU) |
| `merge.CheckMergeability` | `{Repo, PRNumber}` | `{State, Snapshot}` | 30s |
| `merge.RebaseWithPolicy` | `{Repo, BranchName, BaseBranch, AllowList}` | `{Outcome: auto-resolved\|halt-non-pointer\|no-rebase-needed, ConflictedPaths, NewSHA}` | 5m |
| `merge.ForcePushWithLease` | `{WorktreeID, RemoteBranch, ExpectedRemoteSHA, NewSHA}` | `{Pushed bool, Reason string}` | 1m |
| `merge.WaitForChecks` | `{Repo, PRNumber, RequiredChecks []string}` | `{AllGreen bool, FailedCheck *string}` | 30m (heartbeat 30s) |
| `merge.MergePR` | `{Repo, PRNumber, Style: "squash"}` | `{MergeSHA, MergedAt, BranchDeleted}` | 1m |
| `merge.DeleteBranch` | `{Repo, BranchName}` | `{Deleted bool, Reason string}` | 30s |
| `merge.EmitQueueTelemetry` | `{SubmissionID, EntryIndex, Step, Reason, Snapshot}` | `{}` | 30s |
| `merge.NotifyOperator` | `{Kind: start\|pause\|complete, SubmissionID, Summary}` | `{}` | 30s |

## State transitions (per-PR)

```text
fetching → classifying
  ↓
classifying → waiting-approval (if governance)
            → rebasing/waiting-checks/merging (else, via mergeability check)
  ↓
waiting-approval → (signal approve) → check mergeability → ...
  ↓
rebasing → pushing (auto-resolved)
        → paused (halt-non-pointer-conflict)
        → check mergeability (no-rebase-needed)
  ↓
paused → (signal resume) → check mergeability
       → (signal abort) → done(aborted)
  ↓
waiting-checks → check mergeability (success)
              → paused (failure or timeout)
  ↓
merging → cleanup (success) → done(merged)
       → paused (merge API error)
  ↓
done(merged | skipped | aborted | failed)
```

Every transition emits one OTLP event (FR-024).

## Validation rules (consolidated)

1. **Submission-time validation** (in CLI before `ExecuteWorkflow`):
   - All `(Repo, PRNumber)` pairs are unique within the submission.
   - All `DependsOn` indices reference earlier positions.
   - No dependency cycles (no cycles possible because indices are forward-only, but we still verify in case of malformed YAML).
   - At least one entry.
   - All `Repo` values match `{owner}/{name}` format.
   - All `Repo` values have credentials available (verified by `gh auth status -h <owner>` or equivalent).
2. **Per-PR runtime validation** (in `PRMergeWorkflow`):
   - PR exists and is OPEN at fetch time (Closed → skip).
   - `ExpectedClass` (if set) does not relax governance.
3. **Pre-merge validation** (in `merge_pr` activity):
   - Branch SHA matches the SHA we last force-pushed (lease check).
   - Required checks are SUCCESS or NEUTRAL.
