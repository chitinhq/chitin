# Contract: Parity Corpus File Format

**Feature**: 086-event-hash-consolidation · **Date**: 2026-05-22

The parity corpus is the shared fixture both the Go and the TypeScript hash-parity tests
consume. It is the conformance contract for FR-002 and FR-005.

## Location

`go/chainhash/testdata/parity-corpus.json` — co-located with the canonical Go module so the
implementation and its conformance fixtures share one source of truth.

## Format

A JSON array of corpus entries. Each entry:

```json
{
  "name": "deeply-nested-object",
  "event": { "...": "an event map — the input to HashEvent" },
  "expected_hash": "<64-char lowercase hex SHA-256>"
}
```

| Field | Type | Meaning |
|---|---|---|
| `name` | string | Unique, human-readable label for the case (used in test output). |
| `event` | object | The event map passed to `HashEvent` / `hashEvent`. May contain a `this_hash` field — it MUST be excluded by the implementation. |
| `expected_hash` | string | The byte-identical hash every implementation MUST produce for `event`. For an entry that exercises an unsupported value type, this field is omitted and `expected_error: true` is set instead. |
| `expected_error` | bool (optional) | Present and `true` when `event` contains an unsupported value type; every implementation MUST return an error and produce no hash. |

## Required entries (one per spec "Edge Cases" item, plus ordinary cases)

| `name` (suggested) | Exercises |
|---|---|
| `ordinary-event` | A typical, valid event with envelope + payload. |
| `historical-chain-record` | An event shaped like a real stored chain record — anchors FR-004. |
| `empty-payload` | An event whose `payload` is `{}` / null envelope fields. |
| `deeply-nested-object` | Multi-level nesting — key ordering at every depth. |
| `unicode-strings` | Non-ASCII / Unicode string values in payload and envelope. |
| `numeric-extremes` | Large integers and floating-point values. |
| `unsupported-type` | A value type outside the permitted set — `expected_error: true`. |

## Authoring rule

`expected_hash` is generated by running the corpus through the canonical TypeScript
`hashEvent` (`libs/contracts/src/hash.ts`) — the cross-language reference of record (FR-009).
The Go corpus test and the cross-language vitest test then both assert against these values.
A corpus entry is added only when a new edge case is identified; it is never edited to make
a failing test pass.
