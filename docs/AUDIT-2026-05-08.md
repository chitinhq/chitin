# Chitin Docs Audit — 2026-05-08

Comprehensive audit of all documentation files against the post-cull codebase (kernel + plugins + data scope, per `docs/decisions/2026-05-06-chitin-scope-narrow-to-kernel.md` and the 2026-05-08 cull decisions).

---

## Executive Summary

The core authoritative docs (README, roadmap, thesis, architecture, layer-contracts, operating-model, driver-conformance, governance-setup) are in good shape post-cull. The 2026-05-08 rewrite was thorough — these files correctly reflect "kernel + plugins + data, not orchestration." The cull decision docs themselves are excellent.

The main issues fall into three buckets:

1. **Superseded specs not moved** — 6 spec/plan/design docs that the cull renders obsolete still sit in active directories (`specs/`, `plans/`, `design/`) without superseded headers or redirects.
2. **Two runbooks with heavy stale references** — `local-qwen-stack.md` and `local-glm-flash-stack.md` reference `apps/runner/src/`, `dispatcher.ts`, `swarm-backlog.md`, and `tier-router-with-advisor-consultation`, all of which are deleted.
3. **Two core docs with minor factual drift** — `event-model.md` says `action_type` is "6 classes" (it's now a much richer closed enum), and `architecture.md` lists a `Souls` layer that is tagged "historical analytics/reference artifact" in the operating model but receives no such qualification in architecture.

Everything else is either correctly stale-marked, correctly updated, or historical-but-harmless.

### M7. `docs/event-model.md` — action_type description says "6 classes" but the actual enum has ~40 values

The actual `internal/gov/action.go` has a rich closed enum with ~40 specific action types (`file.read`, `file.write`, `git.push`, `kanban.call`, `hermes.process`, etc.). The "6 classes" language was true of an earlier design but is now inaccurate.

**Fix:** Replace the "6 classes" description with the actual closed-enum concept. Reference `docs/governance-setup.md` or `internal/gov/action.go` for the full list.
